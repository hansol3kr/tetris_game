# Blockfall 실행 방법

Godot 4.3 + C#(.NET 8)으로 만든 네온 낙하 블록 퍼즐 게임입니다.
이 문서는 **처음 받은 사람이 게임을 실행하기까지** 필요한 최소 절차만 정리합니다.
(상세 빌드·배포 가이드는 [`docs/BUILD.md`](docs/BUILD.md) 참고)

> **한 번에 실행:** 도구가 설치돼 있다면 아래 스크립트 하나로 빌드+실행됩니다.
> ```bash
> ./run.sh            # 코어 빌드 후 게임 실행
> ./run.sh --test     # 코어 테스트만 (Godot 불필요)
> ./run.sh --editor   # Godot 에디터로 열기
> ```
> Godot이 PATH에 없으면 `GODOT=/경로/Godot_v4.3-stable_mono_linux.x86_64 ./run.sh` 처럼 지정하세요.

---

## 1. 준비물 (설치)

| 도구 | 버전 | 비고 |
| --- | --- | --- |
| **.NET SDK** | **8.0.x** | 코어 라이브러리·테스트와 Godot C# 빌드에 필요 |
| **Godot 엔진** | **4.3 — .NET(Mono) 버전** | ⚠️ 반드시 **".NET / Mono" 빌드**를 받아야 함. 일반(Standard) 빌드는 C#을 못 씀 |

- .NET 8 SDK: <https://dotnet.microsoft.com/download/dotnet/8.0>
- Godot 4.3 .NET: <https://godotengine.org/download>

> **이 PC(Ubuntu 26.04)에는 이미 설치·설정 완료:**
> - .NET 8 SDK → `~/.dotnet` (apt엔 .NET 10만 있어 홈 디렉터리에 설치)
> - Godot 4.3 mono → `~/.local/godot/…`
>
> `run.sh`가 두 경로를 자동으로 찾으므로 **`./run.sh`만 실행하면 됩니다.**
> 터미널에서 직접 `dotnet`을 쓰려면 아래를 셸에 추가하세요(현재 시스템 `dotnet`은 깨져 있음):
> ```bash
> export DOTNET_ROOT="$HOME/.dotnet"
> export PATH="$HOME/.dotnet:$PATH"
> ```

설치 확인:

```bash
dotnet --info      # 8.0.x SDK가 보여야 함
godot --version    # 4.3.x.mono 로 보여야 함 (.mono 여부 중요)
```

## 2. 코어 테스트만 실행 (Godot 없이 확인)

게임 규칙 엔진(`Blockfall.Core`)은 순수 C#이라 Godot 없이도 검증됩니다.
저장소 루트에서:

```bash
dotnet test Blockfall.sln
```

**131개 테스트 통과**가 나오면 정상입니다.

---

## 3. 게임 실행 (Godot 에디터)

1. **Godot 4.3 .NET 에디터**를 실행합니다.
2. **Import(가져오기)** → `game/project.godot` 파일을 선택합니다.
3. 처음 열면 C# 솔루션이 자동으로 빌드됩니다.
   - 만약 빌드가 안 되거나 어셈블리 오류가 나면, 루트에서 한 번 수동 빌드하세요:
     ```bash
     dotnet build game/Blockfall.csproj
     ```
4. 에디터 상단의 **▶ 재생(Play)** 버튼 또는 **F5**를 누르면 게임이 실행됩니다.
   - 시작 씬은 `res://scenes/Main.tscn` 입니다.

> 에디터/데스크톱 개발 실행에서는 광고·스토어 기능이 꺼진 `NullPlatform`으로
> 동작하므로, 별도 플러그인 없이 바로 플레이할 수 있습니다.

### 창 없이 빌드만 확인 (선택)

```bash
godot --headless --path game --build-solutions --quit
```

## 5. 게임 모드

| 모드 | 목표 |
| --- | --- |
| **Daily Challenge** | 그날 모두에게 동일한 시드로 2분 점수 어택 (리더보드용) |
| **Marathon** | 라인을 지우며 레벨 15까지 도달 |
| **Sprint 40** | 40라인을 최대한 빠르게 클리어 (기록 경쟁) |
| **Ultra 2:00** | 2분 동안 최대한 높은 점수 |
| **Zen** | 시간·목표 없는 무한 릴랙스 플레이 |
| **Dig Race** | 미리 쌓인 가비지 10줄을 최대한 빠르게 파내기 (기록 경쟁) |
| **Survival** | 가비지가 점점 빠르게 차오름 — 라인 클리어(공격)로 상쇄하며 버티기 |
| **Master 20G** | 즉시 낙하 + 짧은 락 + 올스핀 채점 (고난도 실행력) |

> 메인 메뉴에서 **DAILY CHALLENGE / SETTINGS**(음량·고스트·색맹 팔레트·**피네스 미터**·화면 흔들림·DAS/ARR),
> **모디파이어**(NO HOLD / NO GHOST / LOW-G / BLITZ / HARD LOCK / ALL-SPIN / INVISIBLE — 아무 모드에나 중첩),
> **시드 입력**(시드 코드나 아무 단어)을 사용할 수 있습니다. 결과 화면의 **REPLAY SEED**로 같은 판을 재도전할 수 있습니다.
>
> **피네스 미터**: 각 조각을 "최소 키입력"과 비교해 실시간 실력(퍼센트 + F→S 랭크)과 클린 배치 스트릭을 보여줍니다.
> **INVISIBLE**: 쌓인 블록이 보이지 않습니다 — 기억으로 플레이하고, 조각이 놓일 때마다 잠깐 보입니다.

## 6. 자주 겪는 문제

- **에디터에 C# 빌드 옵션이 없음 / C# 사용 불가**
  → 일반 Godot을 설치한 것. **.NET/Mono 빌드**로 다시 설치하세요.
- **`Blockfall.Core` 어셈블리를 찾을 수 없음**
  → `dotnet build game/Blockfall.csproj` 실행, .NET 8 SDK가 `PATH`에 있는지 확인.
- **테스트가 안 돌아감**
  → `dotnet --info`로 8.0.x SDK가 잡히는지 확인 후 `dotnet test Blockfall.sln`.
- **`X11 Display is not available` / `Can't connect to a Wayland display` → 세그폴트**
  → 화면이 없는 세션(주로 **SSH 원격 접속**)에서 GUI를 띄우려 해서 납니다.
  게임 창을 보려면 **이 PC 모니터 앞에서 데스크톱에 로그인**해야 합니다(모니터에 GDM 로그인 화면만
  떠 있으면 아직 로그인 전). 로그인한 뒤에는:
  - 데스크톱 안의 터미널에서 `./run.sh`, **또는**
  - SSH 창에서 그대로 `./run.sh` — `run.sh`가 그 그래픽 세션(`DISPLAY`/`WAYLAND_DISPLAY`)을
    **자동으로 찾아** 모니터에 띄웁니다.

  SSH에서 확인만 할 때는 `./run.sh --test` 또는 `./run.sh --headless`.
  (디스플레이를 못 찾으면 `run.sh`는 세그폴트 대신 안내 메시지를 내고 멈춥니다.)
